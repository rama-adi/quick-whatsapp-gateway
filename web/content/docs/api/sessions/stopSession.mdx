      - content: >-
          Disconnects a session from WhatsApp **without** unlinking the device.


          Requires the `manage` capability.


          The attached number stays paired, so you can `:start` it again later
          without re-pairing. To unlink the device entirely, use `:logout`
          instead.


          **Idempotency.** Stopping an already-stopped session is a harmless
          no-op. The response returns the refreshed session so you can see its
          new status.


          **Errors.** `not_found` (404) — no such session in your organization.
          `unauthorized` (401) / `forbidden` (403) — credential / capability
          failures.


          Returns **200** with the refreshed session.
