---
title: Delete a webhook
description: >-
  Delete the webhook identified by `id`. Requires the `manage` capability, and
  the webhook must belong to the caller's organization.


  After deletion no further events are delivered to its url; any deliveries
  already pending retry stop. The deletion is permanent — there is no
  soft-delete (to pause deliveries without removing the config, set `active:
  false` via update instead).


  On success returns `204` with an empty body. Errors: `404 not_found` if no
  webhook with that id is owned by the caller's organization; `403 forbidden` if
  the caller lacks the `manage` capability.
