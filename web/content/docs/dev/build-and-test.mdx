---
title: Build and test gates
description: The commands each half of the repo must pass before a change lands.
---

Each half has its own gate, and **both must stay green at every committed step**.

## Gateway (Go)

From the repo root:

```sh
go build ./... && go test ./...
```

The build and tests run with `CGO_ENABLED=0` (the pure-Go SQLite driver). The
trust seam is locked by contract tests in `internal/authz/` that mint a real
better-auth JWT and api-key and verify them in the gateway — run those alone with:

```sh
CGO_ENABLED=0 go test ./internal/authz/... ./internal/controlbus/...
```

Lint with `make lint` (`golangci-lint run`).

## Frontend (`web/`)

From `web/`:

```sh
pnpm build && pnpm typecheck && pnpm test
```

| Command | What it does |
|---|---|
| `pnpm build` | Vite build → `.output` (the Nitro server bundle). Also runs the TanStack Router code generator, which produces `app/routeTree.gen.ts`. |
| `pnpm typecheck` | `tsc --noEmit`. It consumes the generated route tree, so run a build (or `pnpm dev`) once on a fresh checkout before typechecking. |
| `pnpm test` | Vitest — covers the NDJSON parser and the `cacheBridge` event reducer. |

## Order matters on a fresh checkout

`tsc` depends on the generated `app/routeTree.gen.ts`, so on a clean clone run
`pnpm build` (or start `pnpm dev`) **before** `pnpm typecheck` — otherwise the
typecheck fails on a missing route tree, not on a real type error.
