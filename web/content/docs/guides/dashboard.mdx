---
title: The dashboard
description: Admin and user surfaces, the read-only viewer, and contacts.
---

The dashboard is where people sign in to pair numbers, create keys and webhooks,
and watch what's happening. What you see depends on who you are: a normal user
works inside their organizations; the platform super admin also gets
oversight across all of them.

## User surface

What every signed-in user gets, scoped to their **active organization** (switch
orgs from the org switcher):

- **Sessions** — the org's WhatsApp numbers. Create them, start/stop them, fetch a
  QR or pairing code, and watch status change live. See
  [connect a number](/docs/guides/connect-a-number).
- **API keys** — create and revoke the org's keys. See
  [API keys](/docs/guides/api-keys).
- **Webhooks** — register and edit where the org's events get pushed. See
  [receive events](/docs/guides/receiving).
- **Viewer** and **Contacts** — below.

## Admin surface

The platform super admin (the first account created) gets everything a user gets,
plus oversight that crosses organizations:

- **Users** — list people, change roles, ban or unban, and impersonate for support.
- **All sessions** — every WhatsApp session across **all** organizations and their
  status, each labeled with the gateway it lives on.
- **The admin number** — the pairing code for the deployment's
  [admin number](/docs/guides/connect-a-number#the-admin-number), so it can be linked
  without console access.

## The viewer

A **read-only** window into a session's WhatsApp: its chats and the message
timeline, updating live as messages arrive. It is for watching, not replying —
sending happens through the [API](/docs/guides/sending). Messages with attachments
show a placeholder rather than the file, since this release does not download media.

## Contacts

A searchable list of the people and groups a session has seen, built from incoming
messages. Open a contact to see their direct chat and the groups you share with
them, including the name they use per group. It's a way to find a recipient's JID
and confirm who you're talking to before you send.
